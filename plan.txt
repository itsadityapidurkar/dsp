==================================================
RESUME ANALYZER: BACKEND LOGIC
==================================================

Implement a clear, explainable, deterministic pipeline.

Step 1: extract resume text
- PDF parser
- DOCX parser

Step 2: preprocess text
- lowercase
- normalize whitespace
- remove noise where helpful

Step 3: extract skills
- use keyword matching against a normalized skill dictionary derived from the unified job market data
- support normalization like:
  - js -> JavaScript
  - py -> Python only if safe
  - power bi -> Power BI
- avoid overcomplicated NLP for v1

Step 4: infer candidate profile attributes where possible
- candidate skills
- candidate likely experience signals
- optional education extraction
- optional certifications/projects extraction if straightforward

Step 5: compute role match scores
For each role:
- derive representative required skills from job market data
- compare candidate skills against role skills
- compute score using a weighted formula

Recommended scoring approach:
- 70% skill overlap
- 20% experience alignment
- 10% category/domain alignment

Example:
match_score =
  0.70 * skill_overlap_score +
  0.20 * experience_alignment_score +
  0.10 * category_alignment_score

If category inference is unavailable, redistribute weight to skill overlap.

Step 6: rank roles
- return top 3 to 5 roles

Step 7: compute skill gap
- missing_skills = target_role_skills - candidate_skills

Step 8: generate suggestions
- based on highest-value missing skills

Step 9: use Gemini only for explanation layer
- summary
- encouragement
- concise recommendations
- short role summaries
- final wording polish

Do NOT rely on Gemini for primary scoring.
